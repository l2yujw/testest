name: CI/CD Pipeline # 워크플로우의 이름을 정의합니다. 이 이름은 GitHub UI에서 워크플로우를 식별하는 데 사용됩니다. on: # 워크플로우가 트리거되는 이벤트를 정의합니다. 다양한 이벤트(예: 푸시, PR, 스케줄 등)를 지정할 수 있습니다. push: branches: - main pull_request: branches: - main jobs: # 실행할 작업(job)을 정의합니다. 각 작업은 독립적으로 실행될 수 있으며, 여러 개의 작업을 정의할 수 있습니다. build: runs-on: ubuntu-latest Env: # 환경 변수를 정의합니다. 이 변수는 워크플로우 전체에서 사용될 수 있습니다. NODE_ENV: production defaults: # 모든 작업에 대해 기본값을 설정할 수 있습니다. 일반적으로 runs나 steps에 적용할 수 있는 기본 설정을 포함합니다. runs: on: ubuntu-latest concurrency: # 동시에 실행할 수 있는 작업의 수를 제한합니다. 특정 워크플로우의 인스턴스가 동시에 실행되지 않도록 설정할 수 있습니다. group: ${{ github.ref }} cancel-in-progress: true permissions: # GitHub Actions가 리포지토리의 리소스에 접근할 수 있는 권한을 설정합니다. 이를 통해 필요한 최소 권한만 부여할 수 있습니다. contents: read issues: write steps: # 각 작업에서 실행할 단계(step)를 정의합니다. 각 단계는 독립적인 작업을 수행합니다. - name: Checkout code uses: actions/checkout@v2 runs-on: ubuntu-latest # 해당 작업이 실행될 환경(런너)를 정의합니다. 예를 들어, ubuntu-latest, windows-latest 등이 있습니다. needs: build # 특정 작업이 완료된 후에만 실행되도록 의존성을 설정합니다.
